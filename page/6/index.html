Git-基础学习

迷心兔

VCS：版本控制系统 【保存所有修订版本，缺点：一不小心会写错文件或者覆盖意想外的文件】
CVCS：集中化的版本控制系统 【让在不同系统上的开发者协同工作，缺点：整个项目的历史记录被保存在单一位置】
DVCS：分布式版本控制系统 【解决以上两个问题】

$ git init

初始化仓库

$ git status

检查当前文件状态

$ git status -s

1.新添加的未跟踪文件前面有 ?? 标记
2.新添加到暂存区中的文件前面有 A 标记
3.修改过的文件前面有 M 标记

$ echo ‘My Project’ > Test.txt

创建一个新的 Test.txt文件,你将看到一个新的未跟踪文件

$ git add Test.txt

跟踪新的 Test.txt文件

$ git add .

跟踪新的 多个文件

$ git clone git@github.com:mixintu/Test.git 【mixintu：自定义名】

克隆现有的仓库

$ cat .gitignore

忽略文件

$ git diff

查看尚未暂存的文件更新了哪些部分 【默认是第一个修改的文件】

$ git diff –staged

查看已暂存的-将要添加到下次提交里的内容 【最后一个修改的文件】

$ git diff –cached

查看已经暂存起来的变化 【已修改的全部文件】

$ git commit -m “提交更新的说明”

提交更新

git push origin master // $ git push [remote-name] [branch-name]

上传至远程仓库

$ git rm Test.txt

移除文件

$ git rm –cached Test.txt

移除文件,但仍然希望保留在当前工作目录中

$ git rm log/*.log

此命令删除 log/ 目录下扩展名为 .log 的所有文件

$ git rm *~

该命令为删除以 ~ 结尾的所有文件

$ git mv Test02.txt Test03.txt

$ git log

查看提交历史

$ git log -p -2

该选项除了显示基本信息之外，还附带了一次commit的变化。
当进行代码审查，或者快速浏览某个搭档提交的commit所带来的变化的时候，这个参数就非常有用了

$ git log –stat

选项在每次提交的下面列出的所有被修改过的文件，
有多少文件被修改了以及被修改过的文件的某些行被删除或添加了。
在每次提交的最后还有一个总结。

$ git log –pretty=format:”%h - %an, %ar : %s”

使用不同于默认格式的方式展示提交历史

$ git log –pretty=format:”%h %s” –graph

合并历史

$ git log –since=2.weeks

列出最近两周的所有提交历史

$ git reset HEAD Test05.txt

取消暂存的文件

$ git remote -v

仓库服务器的默认名字 和 其对应的 URL

$ git remote add pd git@github.com:mixintu/Test.git

添加远程仓库的连接名

$ git fetch pb // $ git fetch [remote-name]

从pd远程仓库中抓取与拉取

$ git remote show origin // $ git remote show [remote-name]

查看某个远程仓库

$ git remote rename pb paul

远程仓库的重命名

$ git remote rm paul

远程仓库的移除

$ git tag -a v1.2 -m “说明：version 1.2”

附注标签:是存储在 Git 数据库中的一个完整对象。
它们是可以被校验的；
其中包含打标签者的名字、电子邮件地址、日期时间；
还有一个标签信息；
并且可以使用 GNU Privacy Guard （GPG）签名与验证。
通常建议创建附注标签，这样你可以拥有以上所有信息。

$ git tag v1.4-lw

轻量标签:只是想用一个临时的标签，或者因为某些原因不想要保存那些信息，轻量标签也是可用的。

$ git show v1.2

显示出提交信息

$ git tag -a v1.1.0 708654d

后期打标签

$ git push origin v1.2

推送一个【未推送的本地标签】到远程

git push origin –tags

推送全部【未推送的本地标签】到远程

$ git tag -d v1.2

删除本地标签

$ git push origin :refs/tags/v1.2

删除本地标签了，在执行该句，删除远程tag

$ git branch testing

创建一个名为testing的分支

$ git log –oneline –decorate

查看各个分支当前所指的对象

$ git checkout testing

切换到testing分支上

$ git log –oneline –decorate –graph –all

输出你的提交历史、各个分支的指向以及项目的分支分叉情况

$ git checkout -b iss53

创建一个名为iss53的分支，并切换到此分支上

$ git checkout -b hotfix

创建一个名为hotfix的分支，并切换到此分支上

$ git merge hotfix

合并分支

$ git branch -D hotfix

删除hotfix分支

$ git mergetool

合并失败时，适当打开图形化工具来解决冲突

$ git branch –merged

查看当前分支 和 别的分支有过合并

$ git branch –no-merged

查看当前分支 没有 和别的分支有过合并

git ls-remote [remote]

来显式地获得远程引用的完整列表

$ git commit –amend

撤消操作：
1.提交后发现提交信息写错了
2.提交后发现漏掉了几个文件没有添加
3.提交后发现忘记了暂存某些需要的修改-取消暂存的文件
4.提交后发现工作目录中已修改的文件-撤消对文件的修改

$ git push origin –delete testing

jQuery 常用方法手册大全/jQuery事件参考

bind事件方法

功能：为元素添加事件【语法一】
语法：$(selector).bind(event,data,function)
参数：

event：一个或多个事件
data：可选。规定传递到函数的额外数据。
function：必需。规定当事件发生时运行的函数。

unbind事件方法

功能：移除被选元素的事件处理程序
语法：$(selector).unbind(event,function)
参数：

event：可选。规定删除元素的一个或多个事件
function：可选。规定从元素的指定事件取消绑定的函数名

bind事件方法

功能：将事件和函数绑定到元素上【语法二】
语法：$(selector).bind(event,data,function)
参数：

{event:function, event:function, …}：必需。规定事件映射，其中包含一个或多个添加到元素的事件，以及当事件发生时运行的函数。

blur事件方法

功能：当元素失去焦点时会触发
语法：$(selector).blur(function)
参数：

function：可选。规定当 blur 事件发生时运行的函数。

change事件方法

功能：在某个域被使用或改变时会触发【语法一】
语法：$(selector).change()
参数：无

注意：该事件仅适用于文本域（text field），以及 textarea 和 select 元素

change事件方法

功能：将函数绑定到 change 事件【语法二】
语法：$(selector).change(function)
参数：

function 可选。规定当 change 事件发生时运行的函数。
注意：该事件仅适用于文本域（text field），以及 textarea 和 select 元素

one事件方法

功能：根据某元素为子元素添加一个或多个事件【每个事件只能执行一次】
语法：$(selector).one(event,data,function)
参数：

event：必需。规定添加到元素的一个或多个事件
data：可选。规定传递到函数的额外数据
function：必需。规定当事件发生时运行的函数。

delegate事件方法

功能：根据某元素为子元素添加一个或多个事件【每个事件可以多次执行】
语法：$(selector).delegate(childSelector,event,data,function)
参数：

childSelector：必需。规定要附加事件处理程序的一个或多个子元素。
event：必需。规定附加到元素的一个或多个事件。
data：可选。规定传递到函数的额外数据。
function：必需。规定当事件发生时运行的函数。

undelegate事件方法

功能：删除由 delegate() 方法添加的一个或多个事件处理程序
语法：$(selector).undelegate(selector,event,function)
参数：

selector：可选。规定需要删除事件处理程序的选择器
event：可选。规定需要删除处理函数的一个或多个事件类型。
function：可选。规定要删除的具体事件处理函数。

live事件方法

功能：添加或附加一个或多个事件
语法：$(selector).live(event,data,function)
参数：

event：必需。规定附加到元素的一个或多个事件
data：可选。规定传递到该函数的额外数据。
function：必需。规定当事件发生时运行的函数。

die事件方法

功能：移除所有通过live()方法向指定元素添加的一个或多个事件
语法：$(selector).die(event,function)
参数：

event：必需。规定要移除的一个或多个事件处理程序
function：可选。规定要移除的特定函数

toggle事件方法

功能：默认是执行hide()与show() 方法
语法：$(selector).toggle(speed,callback)
参数：

speed：毫秒（比如 1500），”slow”，”normal”，”fast”
callback：回调函数

toggle事件方法

功能：自定义事件函数
语法：$(selector).toggle(function1(),function2(),functionN(),…)
参数：

function1() 必需。规定当元素在每偶数次被点击时要运行的函数
function2() 必需。规定当元素在每奇数次被点击时要运行的函数
functionN(),… 可选。规定需要切换的其他函数

trigger事件方法

功能：触发被选元素的指定事件类型【有冒泡事件】
语法：$(selector).trigger(event,[param1,param2,…])
参数：

event：必需。规定指定元素要触发的事件
[param1,param2,…]：可选。传递到事件处理程序的额外参数

triggerHandler事件方法

功能：触发被选元素的指定事件类型【不会产生事件冒泡】
语法：$(selector).triggerHandler(event,[param1,param2,…])
参数：

event：必需。规定指定元素要触发的事件
[param1,param2,…]：可选。传递到事件处理程序的额外参数
