Git-基础学习
发表于 2020-03-04| 更新于 2020-03-06
阅读量:

迷心兔

VCS：版本控制系统 【保存所有修订版本，缺点：一不小心会写错文件或者覆盖意想外的文件】
CVCS：集中化的版本控制系统 【让在不同系统上的开发者协同工作，缺点：整个项目的历史记录被保存在单一位置】
DVCS：分布式版本控制系统 【解决以上两个问题】

$git init

初始化仓库

$ git status

检查当前文件状态

$ git status -s

1.新添加的未跟踪文件前面有 ?? 标记
2.新添加到暂存区中的文件前面有 A 标记
3.修改过的文件前面有 M 标记

$ echo ‘My Project’ > Test.txt

创建一个新的 Test.txt文件,你将看到一个新的未跟踪文件

$ git add Test.txt

跟踪新的 Test.txt文件

$ git add .

跟踪新的 多个文件

$ git clone git@github.com:mixintu/Test.git 【mixintu：自定义名】

克隆现有的仓库

$ cat .gitignore

忽略文件

$ git diff

查看尚未暂存的文件更新了哪些部分 【默认是第一个修改的文件】

$ git diff –staged

查看已暂存的-将要添加到下次提交里的内容 【最后一个修改的文件】

$ git diff –cached

查看已经暂存起来的变化 【已修改的全部文件】

$ git commit -m “提交更新的说明”

提交更新

git push origin master // $ git push [remote-name] [branch-name]

上传至远程仓库

$ git rm Test.txt

移除文件

$ git rm –cached Test.txt

移除文件,但仍然希望保留在当前工作目录中

$ git rm log/*.log

此命令删除 log/ 目录下扩展名为 .log 的所有文件

$ git rm *~

该命令为删除以 ~ 结尾的所有文件

$ git mv Test02.txt Test03.txt

$ git log

查看提交历史

$ git log -p -2

该选项除了显示基本信息之外，还附带了一次commit的变化。
当进行代码审查，或者快速浏览某个搭档提交的commit所带来的变化的时候，这个参数就非常有用了

$ git log –stat

选项在每次提交的下面列出的所有被修改过的文件，
有多少文件被修改了以及被修改过的文件的某些行被删除或添加了。
在每次提交的最后还有一个总结。

$ git log –pretty=format:”%h - %an, %ar : %s”

使用不同于默认格式的方式展示提交历史

$ git log –pretty=format:”%h %s” –graph

合并历史

$ git log –since=2.weeks

列出最近两周的所有提交历史

$ git reset HEAD Test05.txt

取消暂存的文件

$ git remote -v

仓库服务器的默认名字 和 其对应的 URL

$ git remote add pd git@github.com:mixintu/Test.git

添加远程仓库的连接名

$ git fetch pb // $ git fetch [remote-name]

从pd远程仓库中抓取与拉取

$ git remote show origin // $ git remote show [remote-name]

查看某个远程仓库

$ git remote rename pb paul

远程仓库的重命名

$ git remote rm paul

远程仓库的移除

$ git tag -a v1.2 -m “说明：version 1.2”

附注标签:是存储在 Git 数据库中的一个完整对象。
它们是可以被校验的；
其中包含打标签者的名字、电子邮件地址、日期时间；
还有一个标签信息；
并且可以使用 GNU Privacy Guard （GPG）签名与验证。
通常建议创建附注标签，这样你可以拥有以上所有信息。

$ git tag v1.4-lw

轻量标签:只是想用一个临时的标签，或者因为某些原因不想要保存那些信息，轻量标签也是可用的。

$ git show v1.2

显示出提交信息

$ git tag -a v1.1.0 708654d

后期打标签

$ git push origin v1.2

推送一个【未推送的本地标签】到远程

git push origin –tags

推送全部【未推送的本地标签】到远程

$ git tag -d v1.2

删除本地标签

$ git push origin :refs/tags/v1.2

删除本地标签了，在执行该句，删除远程tag

$ git branch testing

创建一个名为testing的分支

$ git log –oneline –decorate

查看各个分支当前所指的对象

$ git checkout testing

切换到testing分支上

$ git log –oneline –decorate –graph –all

输出你的提交历史、各个分支的指向以及项目的分支分叉情况

$ git checkout -b iss53

创建一个名为iss53的分支，并切换到此分支上

$ git checkout -b hotfix

创建一个名为hotfix的分支，并切换到此分支上

$ git merge hotfix

合并分支

$ git branch -D hotfix

删除hotfix分支

$ git mergetool

合并失败时，适当打开图形化工具来解决冲突

$ git branch –merged

查看当前分支 和 别的分支有过合并

$ git branch –no-merged

查看当前分支 没有 和别的分支有过合并

git ls-remote [remote]

来显式地获得远程引用的完整列表

$ git commit –amend

撤消操作：
1.提交后发现提交信息写错了
2.提交后发现漏掉了几个文件没有添加
3.提交后发现忘记了暂存某些需要的修改-取消暂存的文件
4.提交后发现工作目录中已修改的文件-撤消对文件的修改

$ git push origin –delete testing

删除远程分支
